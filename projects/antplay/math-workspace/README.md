## Claim Boundary

可保守說:

- 維護過 math workspace KB、module index、驗證流程、分支 / 推送 / debug 工具紀錄。
- 可作工作方法 supporting evidence：跨 math repo code reading、GDD / RTP / reel strip / debug flow 梳理。
- 可說明 `math-workspace` 如何支撐 `star-math` flow 的開發 / 驗證閉環：GDD / 規格書、相似成熟底包、固定盤面、result contract、RTP / reel strip、optimizer / final valid 與 KB 回填。

不可誇大:

- 不寫 production service 開發。
- 不反向包裝成每個 math module 都真實開發過。
- 不寫完整遊戲數學 / RTP owner。
- 不把 `sdt-lab-math` 的安全 / 重構 / 單測 lab 經驗，直接升級成全部 `*-math` 的預設開發規則或完整測試平台 owner。
