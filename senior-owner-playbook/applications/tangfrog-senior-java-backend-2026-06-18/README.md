## 結論

這份 JD 可投，但優先順序低於同公司 Backend Team Lead 缺。

建議定位：

```text
Senior Java Backend / Game Backend Engineer
```

主打：

- 遊戲 / 博弈後端 production flow。
- Java / Spring / MyBatis / MySQL / Redis / RabbitMQ。
- Provider gateway、payment provider、wallet / bet-settle。
- 系統維運、效能排查、log / DB / Redis / MQ flow reconstruction。
- 技術文件、開發標準、AI-assisted engineering workflow。

降調：

- 這份不需要管理責任，所以不要使用 Team Lead 包的強帶人定位。
- 薪資上限 `100,000` 偏保守，可作 market check / 保底機會，不當最高優先。
- Netty / Elasticsearch 若沒有明確實戰，不放主軸，只列可補強。
- 架構設計只寫參與 / 能分析 / 能提出方案，不寫完整架構師或完整系統 owner。

## 與 Team Lead 缺的差異

| 職缺 | 建議定位 | 優先順序 | 薪資策略 |
| --- | --- | --- | --- |
| Backend Team Lead (Java) | Senior Java Backend / Hands-on Backend Tech Lead Candidate | 較高，用來測上限 | 120,000-140,000 |
| 資深JAVA後端工程師 | Senior Java Backend / Game Backend Engineer | 可投，偏 market check / 保底 | 100,000-110,000 |

## 投遞策略

用 `08` A 版通用高交易 Senior Java Backend / Platform Backend 為底，前移遊戲後端、provider、wallet、RabbitMQ、Redis、MySQL、系統維運、技術文件與 AI-assisted workflow。不要把這份投成 Lead，也不要為了 JD 的「架構設計」把自己包裝成完整 architect。
